# 《死亡威胁》课程 P1：lv_0_20230416155620 🎮

在本节课中，我们将学习如何通过修改游戏内存中的特定数值，实现“无限生命”的效果。这是一种常见的内存修改技术，适用于单机游戏。

## 概述

我们将使用一个名为“Cheat Engine”的工具，通过扫描和锁定内存地址来修改游戏中的生命值。整个过程分为几个关键步骤。

## 工具准备

首先，你需要准备以下工具：
*   Cheat Engine（内存扫描与修改工具）
*   目标游戏（本节课以演示游戏为例）

## 步骤详解

上一节我们介绍了所需的工具，本节中我们来看看具体的操作流程。

### 1. 初始扫描

打开Cheat Engine和目标游戏。在Cheat Engine中选择游戏进程。假设游戏初始生命值为100，我们在扫描类型中选择“精确数值”，数值输入“100”，然后执行首次扫描。

### 2. 筛选地址

返回游戏，让角色受到伤害，使生命值发生变化（例如变为90）。在Cheat Engine的数值框中输入新的生命值“90”，点击“再次扫描”。这会筛选出数值发生变化的地址。

### 3. 定位与确认

重复上一步骤，直到左侧的地址列表仅剩少数几个地址。通过尝试修改这些地址的数值，并在游戏中验证生命值是否随之改变，从而找到存储生命值的正确地址。

以下是确认地址后的关键操作：
*   将该地址添加到下方的地址列表。
*   手动修改该地址的数值以进行测试。
*   如果修改成功，则激活“锁定”功能，将其数值固定（例如锁定为100），即可实现无限生命。

其核心逻辑可以用一个简单的伪代码表示：
```cpp
// 假设找到的生命值内存地址
int* healthAddress = 0x12345678;
// 锁定生命值
*healthAddress = 100;
```

### 4. 生成脚本（进阶）

对于高级应用，可以生成自动执行该操作的脚本。在地址上右键，选择“自动汇编”，然后使用模板“代码注入”。在注入的代码中编写指令，例如：
```
mov [eax+04], 100 // 将生命值设置为100
```
这样每次游戏尝试更新生命值时，都会被我们的代码覆盖。

## 总结

本节课中我们一起学习了使用Cheat Engine进行内存修改的基本流程：从初始扫描、筛选变化地址，到定位确认并锁定数值。最后，我们还了解了如何通过生成自动汇编脚本实现自动修改。请记住，此方法仅适用于单机游戏的学习与研究。
