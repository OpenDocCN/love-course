# 课程名称：得意苑 🏡

在本节课中，我们将学习如何构建一个名为“得意苑”的虚拟空间。我们将从核心概念入手，逐步介绍其构成要素与实现方法。

## 1. 核心概念：空间与对象

上一节我们介绍了课程目标，本节中我们来看看“得意苑”的基础构成。任何虚拟空间都由基本对象和它们之间的关系构成。核心概念可以用一个简单的公式表示：

**得意苑 = { 对象₁, 对象₂, ..., 对象ₙ }**

其中，每个“对象”代表空间中的一个独立元素，例如一栋建筑、一棵树或一个装饰品。

## 2. 构建基础对象

理解了空间由对象集合构成后，我们需要学习如何创建这些基础对象。每个对象都拥有一些基本属性。

以下是定义一个基础对象所需的关键属性列表：

*   **标识符 (ID)**：对象的唯一名称。
*   **类型 (Type)**：对象的类别，例如“建筑”、“植物”、“家具”。
*   **位置 (Position)**：对象在空间中的坐标，通常用 `(x, y, z)` 表示。
*   **状态 (State)**：对象的当前状况，例如“开启”、“关闭”、“生长中”。

在编程中，我们可以用一个类来定义它：

```python
class GardenObject:
    def __init__(self, obj_id, obj_type, position, state):
        self.id = obj_id
        self.type = obj_type
        self.position = position  # 例如 (0, 0, 0)
        self.state = state
```

## 3. 定义对象间交互

对象创建好后，静态的空间缺乏生机。本节我们将探讨如何让对象之间产生互动，这是让“得意苑”活起来的关键。交互通过预定义的规则或动作触发。

以下是几种常见的交互类型：

*   **触发式交互**：当用户点击某个对象时，触发特定效果。
*   **条件式交互**：当满足特定条件（如时间、对象状态）时，自动发生事件。
*   **对象间交互**：两个或多个对象接近时，产生连锁反应。

例如，一个“浇水”动作作用于“植物”对象，可以改变其状态：

```python
def water_plant(plant_object):
    if plant_object.type == "植物":
        plant_object.state = "已浇水"
```

## 4. 整合与呈现

我们已经定义了对象及其交互规则，最后一步是将所有部分整合起来，并呈现给用户。这涉及到管理所有对象的集合，并处理它们之间的逻辑。

这个过程可以概括为以下步骤：

1.  **初始化空间**：创建所有对象的实例，并设置初始位置和状态。
2.  **运行主循环**：持续检查用户输入和对象状态，触发相应的交互。
3.  **渲染画面**：根据所有对象的当前属性，在屏幕上绘制出“得意苑”的视觉形象。

一个简化的主循环框架如下：

```python
def main_loop(garden_objects):
    while True:
        # 1. 处理用户输入
        user_action = get_user_input()
        # 2. 更新对象状态（根据交互规则）
        update_objects(garden_objects, user_action)
        # 3. 渲染当前画面
        render_scene(garden_objects)
```

本节课中我们一起学习了构建“得意苑”虚拟空间的完整流程。我们从**空间是对象的集合**这一核心公式出发，逐步学习了如何定义对象属性、设计对象间的交互规则，最终将所有部分整合并呈现出来。掌握这些基础步骤，你就已经迈出了创建自己虚拟世界的第一步。
