# 课程13：时间压力下的决策模拟 🕐

在本节课中，我们将通过一个模拟“时间压力下决策”的场景，来学习如何用编程逻辑描述条件判断与循环计数过程。这个场景基于一个简单的互动：一方给出极短的时间限制要求另一方做出决定。

## 概述

我们将把一段描述性的对话，转化为结构化的逻辑步骤。核心在于理解“计时”和“条件触发”这两个概念。下面，我们将分步拆解这个过程。

## 核心逻辑拆解

上一节我们概述了本课目标，本节中我们来看看如何将文字描述转化为逻辑步骤。整个过程可以分解为以下几个关键环节。

以下是场景中涉及的步骤列表：

1.  设定一个时间限制（例如13秒）。
2.  发起方向接收方提出一个提议（例如“结婚”）。
3.  声明规则：在时限内保持沉默将被视为同意。
4.  开始计时并计数。
5.  在计时过程中或结束后，检查接收方是否有回应。
6.  根据是否有回应，判定结果。

## 关键概念与代码表示

接下来，我们深入看看“计时检查”和“条件判定”这两个核心概念如何用代码表达。

**1. 计时循环**
这模拟了倒计时的过程。我们可以使用一个简单的循环来模拟时间流逝。

```python
time_limit = 13
for second in range(1, time_limit + 1):
    print(f"计数: {second}")
    # 在实际场景中，这里会等待一秒并检查输入
```

**2. 条件判定**
判定结果取决于接收方在时限内是否做出回应。这可以用一个条件语句来表示。

```python
response_received = False # 假设初始没有回应

if not response_received:
    result = "同意"
else:
    result = "拒绝"
```

## 流程整合与总结

最后，我们将上述逻辑整合起来，形成一个完整的模拟流程。首先初始化变量，然后进入计时循环，循环结束后根据条件判定最终结果。

本节课中，我们一起学习了如何将一个带有时间压力的决策场景转化为编程逻辑。我们掌握了用**循环**来模拟计时过程，并用**条件判断**（`if-else`语句）来根据是否接收到回应决定最终结果。这个模型是理解交互式程序决策的基础。
