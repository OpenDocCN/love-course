# 7、chris『红丸会员』：高阶框架打造03吵架中如何掌控框架_note

在本节课中，我们将学习如何在两性争吵中运用高阶框架。吵架本身并不可怕，可怕的是在吵架中丢失了框架。我们将通过具体案例，分析不同框架组合对关系的影响，并找到最健康的应对方式。

---

## 吵架的本质与框架

上一节我们介绍了框架的基本原理，本节中我们来看看它在冲突场景下的具体应用。

在我们的生活中，与伴侣、亲人或朋友发生争吵很常见。因为两个独立的个体，想法和考虑问题的角度不同，产生分歧在所难免。然而，有的情侣吵完架关系更好，有的却逐渐冷淡。

关键在于，吵架中包含了框架的博弈。正确的框架运用，能让感情不会因争吵而伤筋动骨。

---

## 案例分析：一场常见的争吵

我们先来看一段模拟的争吵对话。

> 女：我帮你电话装了定位程序，就查到你在哪里。为什么你装了我不知道？啊？我问你为什么啊？
> 男：那你睡着了就不知道了。
> 女：你是不是搞错了？你是不是追踪我隐私了？
> 男：那你没有做错事，你怕被人查到？你当我是什么？嫌疑犯？恐怖分子？用卫星追踪我？
> 女：你有什么不可以打给我的吗？那你为什么不听电话？
> 男：我的电话和别人交换了，你自己看清楚。你谎话能不能编得连贯一点？你电话不见了，不第一时间跟我说？
> 女：吵吵架你又说这些。不如你先说清楚那女孩是谁。
> 男：我懂不懂？你冤枉我够了没有？你骗我够了没有？我跟你说，你信就信，不信就算。
> 女：我不信。那怎么办？
> 男：那你就没话说。你一辈子都是这种坏态度，做错事都觉得别人对不起你。你现在是不是很恨我？我问你是不是很恨我？
> 女：那两天后还结不结婚？
> 男：见到就结，见不到就不结。
> 女：你自己说的，你记住自己说过什么。

根据第二课学到的框架原理分析，视频中的男女双方都采用了**硬框架**。他们都站在自己的立场，维护自己的利益和感受。在这种场景下，双方对彼此的吸引力不会下降，但**可得性**都会降低。

然而，在长期关系中，并非吸引力主宰一切。长久的高吸引力、低可得性，会让关系陷入冰冷的僵局。生活中常见到，吵架后冷战几天，然后一方去哄另一方，这其实就是最基本的框架运用：**吵架时用硬框架，哄人时用软框架**。在这种软硬结合的推拉中，关系会产生不同走向。

## 四种吵架框架组合

以下是吵架时可能出现的四种框架组合，理解它们对经营关系至关重要。

### 1. 硬框架 → 硬框架
吵架时是硬框架，吵完架依然保持硬框架，等待对方服软来哄你。

*   **公式**：`框架状态 = 硬 -> 硬`
*   这种方式极大满足了你的自尊心，但意味着伴侣的自尊心未被满足。对方愿意先放下面子哄你，是更爱、更包容的表现。此时应及时给予正面反馈。若长期如此，关系会面临极强的不稳定性。

### 2. 硬框架 → 软框架
吵架时是硬框架，但在合适时机转为软框架去哄对方，或让关系恢复正常，就像没吵过架一样。

*   **公式**：`框架状态 = 硬 -> 软`
*   软框架合格呈现的前提是：你已经冷静下来，想清楚了，真心想修复关系。而不是心不甘情不愿地敷衍。你的起心动念（态度）决定了效果。真诚的道歉能被感知，敷衍的道歉只会让对方更生气。

### 3. 软框架 → 硬框架
吵架时为回避冲突而暂时服软（软框架），但事情过后依然我行我素（硬框架）。

*   **公式**：`框架状态 = 软 -> 硬`
*   这种情况很常见，源于息事宁人的习惯。其致命点是，服软的出发点是为了**逃避问题**而非**解决问题**，这是一种虚假的软框架。这样不仅无法拉近关系，反而会让问题在暗处滋长，让对方一次比一次更抓狂。

### 4. 软框架 → 软框架
吵架时是软框架，吵完架后仍是软框架。

*   **公式**：`框架状态 = 软 -> 软`
*   这种模式极易进入“跪舔”的情感经营轨道。时间不用太长，对方就会感到你的窝囊和懦弱，你的**吸引力会迅速流失**。即使你有房有车，对方对你的荷尔蒙系统也会逐渐关闭，这个过程不受理性思维控制。最终关系对双方都是煎熬，且难以挽回。

## 总结与选择

本节课中，我们一起学习了吵架中的四种框架组合。

*   **第一种（硬->硬）**：极易进入高冷状态，感情容易被“冷死”。
*   **第四种（软->软）**：极易进入跪舔状态，感情会被“烫死”。
*   **第三种（软->硬）**：本质是虚假沟通，问题被掩盖却暗中长大，感情会“病死”。
*   **第二种（硬->软）**：是唯一值得我们在生活中思考、学习和运用的健康模式。

至于其中软硬框架的具体比例和更精细的运用，我们将在后续课程中继续探讨。

记住，吵架是一种高能量的沟通方式，它至少能确保对方在真实表达内心。不必害怕吵架，只要心中有爱，并懂得运用正确的框架进行调整，关系就能在冲突中得到淬炼和升华。
