# 课程03：沟通课 丨 成为说话高手 🗣️
## 第26节：活学活用——成为有逻辑的沟通高手 🧩

在本节课中，我们将学习如何将之前课程中介绍的各种逻辑框架灵活应用于实际沟通场景。核心在于理解框架并非一成不变的公式，而是可以根据具体情境进行组合、调整和创新的工具。

不知不觉，我们的课程已接近尾声。最后一节，我想探讨一个贯穿前面所有小节的核心话题：如何更好地应用所学的逻辑框架。

为了能够灵活运用这些框架，我想讲三点。

### 1. 识别场景 🎯

在课程中，我们讲解了许多不同的沟通逻辑与框架。然而，仅仅记住它们是不够的，关键在于用对场景。你需要判断当前是什么场景。

应该对应什么样的沟通逻辑。例如，当前场景是说服对方采纳建议、给予对方反馈，还是希望对方改正某个错误。场景如同开关，只有打开正确的开关。

我们才能取出合适的钥匙，打开沟通之门。

也就是找到合适的逻辑框架。可能有细心的学员已经发现，在某些场景中，所需的沟通逻辑内容是相通的。

其中的一些模块是重合的。例如，我们讲的若干个沟通逻辑框架都会涉及“给予建议”。

也有若干框架会涉及“分析现状”，还有若干框架都会涉及“分析问题原因”。因此，对于同一个场景，可能有多种逻辑或内容都适用。我们在应用时，基于这个场景。

灵活选择适合的内容和逻辑即可。

### 2. 活学活用 🔄

接下来，我们看看如何灵活应用。每个逻辑框架在讲解时，基本都建议你按1234的顺序讲述几个部分，但这并非死板的规定。

例如，四块内容可以组合出不同的顺序。课程中讲解的只是普适性最强、最普遍的一种顺序，或者说在默认状态下这样讲比较可靠、安全、合理。但大家完全可以根据实际情况打乱或调整它们的顺序。

例如，在某一节课讲到的“3-1框架”，原本建议先说愿景，再说分析评估，最后说建议。但有时也可以打乱顺序，比如先说建议，然后说你的分析是什么。说完分析。

最后再说愿景，即可以实现什么。这取决于什么呢？取决于对方更关注什么、想听什么。也就是我们需要判断，在这个沟通场合下，先说什么可能更合适。这要求我们对沟通对象。

有一定的了解。例如，他是喜欢先听行动（怎么做），还是喜欢先听结论（做法能带来什么结果）。再比如，在讲到“提出期许”时，我们提到了“B-I-E-R框架”。

这个框架有四个部分：行为、影响、期望、结果。这个顺序同样可以打乱，甚至只抽取其中的两三个模块来讲。例如，当你和对方关系非常好、信任基础牢固时，就可以直接先说“期望”和“结果”。

也就是直接说你希望对方怎么改，以及希望达到什么结果，这完全可以。在需要时，再补充前面的两个部分。总之，课程中讲解的逻辑是一个参考。大家既可以按原有顺序沟通，也可以灵活变通，不被逻辑框架锁死。

我相信，随着大家对逻辑框架掌握得越来越熟练，也能找到自己认为最合适的逻辑顺序。

### 3. 总结与更新 📈

最后，我们来探讨如何总结与创新。无论我们讲解多少种沟通逻辑。

总会有一些新的情况、新的场景需要我们思考该如何表达。我希望大家学完后，能够自己。

总结出一些梳理沟通逻辑的秘诀。这样可以帮助我们应对课程中未涉及到的场景。当你练习和思考得足够多之后，会发现许多逻辑框架之间其实存在共通之处。

例如，在某些场景下，我们总会谈到未来，然后回到当下要做什么。而在另一些场景下，我们总会谈到一些客观事实，作为建议的基础。还有一些场景下，我们总会谈到对对方的理解等。有了这些基础。

当我们遇到不同的场合和场景时，就可以发明自己的框架。即利用课程中学到的内容模块，对它们进行重新组合，发明出适合自己或特定场景的新逻辑框架。
