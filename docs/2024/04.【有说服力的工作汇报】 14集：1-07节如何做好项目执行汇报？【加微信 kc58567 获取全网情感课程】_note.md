# 课程04：有说服力的工作汇报 - 第07节：如何做好项目执行汇报？🚀

在本节课中，我们将学习如何在项目执行过程中，当遇到计划外的困难或问题时，进行清晰、有效的汇报，以推动项目继续前进。

上一节我们介绍了项目启动阶段的汇报。本节中，我们来看看项目进入执行阶段后，当出现意外情况时该如何汇报。

## 概述：为何需要项目执行汇报？

项目启动时，我们已制定好方案、计划和分工。但项目由人推动，只要涉及人，就存在不确定性。项目周期越长、涉及人员和部门越多，不确定因素也越多。此外，还有外部不可控因素。计划永远无法做到万无一失，我们只能学会应对变化。

## 核心汇报模型

遇到计划外的问题时，推荐使用一个由三部分构成的汇报模型：**情况描述**、**问题对策**以及**推进计划**。

以下是该模型的详细分解。

### 第一部分：情况描述

在直接提出问题前，需先对项目总体情况进行介绍和铺垫。这有助于汇报对象充分理解问题背景。

情况描述包含两个部分：

1.  **项目现状**：描述项目当前进展到了什么程度，已取得哪些成果。
    *   *公式*：`项目现状 = 当前进度 + 已获成果`
2.  **环境条件**：说明项目所处的客观环境与具备的条件。
    *   *公式*：`环境条件 = 内部环境 + 外部环境`

**示例**：
> “项目方案已开始落地，80%的门店完成装修，正在招聘人员，其中两家店人员已到位，可准备开业。”（项目现状）
> “江浙地区连续降雨，门店所在商业街存在安全隐患，所有施工已暂停。”（环境条件）

清晰的情况描述能帮助听众理解问题产生的背景、原因及其严重性。

### 第二部分：问题对策

描述清楚背景后，进入核心的问题与对策部分。

以下是说明问题时的两个关键点：

*   **对照原计划**：明确指出问题如何偏离了原定计划。
    *   *代码示例*: `if (实际进度 < 计划进度) { 报告延期问题 }`
*   **阐明后果**：说明如果问题不解决，将带来什么负面影响。
    *   *代码示例*: `if (问题未解决) { 后果 = 错过关键节点; 影响后续运营 }`

**示例**：
> “关于门店装修延期问题：原计划本月底完成A城市门店装修。受天气影响，进度已落后一周。若问题不解决，将导致门店错过下月购物节开业，使新店宣传效果大打折扣，并影响后续运营。”

提出问题是第一步，紧接着需要给出你的**解决思路**。此时应先提出方向性思路，而非详细计划，目的是让汇报对象进行初步判断。

**思路示例**：
> 思路A：全力追赶装修进度，使项目回归正轨。
> 思路B：以确保按时开业为更大目标，同步推进招聘、物料准备等工作，不局限于解决装修延期。

### 第三部分：推进计划

当解决思路获得认同时，需进一步汇报具体的实施计划。

这部分需要说明：

1.  **具体行动**：为解决上述问题，将采取哪些具体步骤。
2.  **所需支持**：需要哪些额外的资源或配合。

**示例**：
> “具体计划：1. 更换更专业的装修队，将两周工期压缩至一周。2. 即刻启动招聘，借用其他门店进行面试与前期培训，确保装修结束后人员立即到位。”
> “所需支持：1. 因在雨天施工，需更换防水材料，施工方要求支付额外费用。2. 需要其他门店配合招聘与培训工作，需提前协调。”

## 综合案例演练

让我们通过一个完整案例，将三部分串联起来，加深理解。

**场景**：员工小朱负责新产品线上宣传项目，执行中遇到计划外情况。

*   **情况描述**：
    *   *项目现状*：宣传文案与设计已按计划完成；已联系10个宣传渠道，正沟通合作方案。
    *   *环境条件*：不同类型渠道对方案要求不同，小型新媒体平台决策快，传统大平台沟通周期长。

*   **问题对策**：
    *   *问题*：重要宣传渠道X沟通数轮后方案仍无法敲定。
    *   *后果*：若无法与X合作，宣传效果将大打折扣；若一直等待，宣传无法按时上线，将影响后续销售。
    *   *对策思路*：优先保证宣传按时上线。双线并行：继续与X渠道沟通；同时联系备用Y渠道，寻求替代方案。

*   **推进计划**：
    *   *具体行动*：1. 按原计划与其他渠道确认方案，确保按时上线。2. 小王继续跟进X渠道，可适当妥协以促成合作。3. 小李立即联系Y渠道，争取合作。
    *   *所需支持*：1. 为加速推进，需加派一人入组协助小王。2. 可能需要追加部分宣传费用，但为保障执行效果是值得的。

## 总结

本节课我们一起学习了在项目执行遇阻时进行汇报的方法。

我们使用的核心模型是：**情况描述 → 问题对策 → 推进计划**。
*   **情况描述**需说明项目现状与环境条件。
*   **问题对策**需对照计划指出问题、阐明后果，并提出解决思路。
*   **推进计划**需明确具体行动与所需资源支持。

实际工作中的问题可能更复杂，但汇报的结构与方向是一致的。掌握此方法，你能更清晰地进行汇报，从而高效推动项目前进。
