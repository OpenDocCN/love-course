### 概述

本节课程将引导你理解Leon 1.0的基本框架，掌握关键组件的使用方法，并学会如何从零开始搭建一个完整的项目蓝图。

---

### 核心概念：蓝图与组件

上一节我们介绍了项目的基本结构，本节中我们来看看蓝图的核心组成部分。

蓝图的核心是**组件化**设计。每个功能模块都被封装成一个独立的组件，通过预定义的接口进行通信和组合。这可以用一个简单的公式来描述项目的模块化结构：

**项目 = 组件₁ + 组件₂ + … + 组件ₙ**

在代码中，一个基础组件通常这样定义：
```javascript
class BaseComponent {
  constructor(config) {
    this.name = config.name;
  }
  connect() {
    // 连接逻辑
  }
}
```

### 从零开始：环境搭建与初始化

理解了核心概念后，我们需要准备开发环境。以下是搭建开发环境的必要步骤。

首先，确保你的系统已安装Node.js。然后，通过命令行初始化项目。

以下是初始化项目的具体步骤列表：

1.  打开终端或命令行工具。
2.  创建一个新的项目目录：`mkdir my-leon-project`
3.  进入该目录：`cd my-leon-project`
4.  使用npm初始化项目：`npm init -y`
5.  安装Leon核心依赖：`npm install leon-core`

完成以上步骤后，你的项目基础环境就配置好了。

---

### 构建你的第一个模块

环境准备就绪后，我们可以开始构建第一个功能模块。我们将创建一个简单的用户问候模块。

这个模块接收一个用户名作为输入，并返回一句问候语。其功能可以用以下伪代码表示：

**输出 = “你好, ” + 用户名**

以下是创建此模块的步骤：

1.  在项目根目录下创建 `modules` 文件夹。
2.  在 `modules` 文件夹内创建 `greeter.js` 文件。
3.  在 `greeter.js` 中编写模块代码：
    ```javascript
    module.exports = (name) => {
      return `你好，${name}！欢迎使用Leon 1.0。`;
    };
    ```
4.  在主文件中引入并调用这个模块。

现在，你已经成功创建了第一个可工作的功能模块。

---

### 整合与测试：让蓝图运行起来

单个模块构建完成后，下一步是将各个模块整合起来，并进行测试以确保它们能协同工作。

我们需要一个主入口文件来加载和协调所有模块。创建一个 `index.js` 文件，并写入以下内容：

```javascript
const greeter = require('./modules/greeter');

console.log(greeter('开发者'));
```

保存文件后，在终端运行 `node index.js`。如果一切正常，你将看到输出：“你好，开发者！欢迎使用Leon 1.0。”

这表明你的第一个Leon蓝图已经成功运行。
