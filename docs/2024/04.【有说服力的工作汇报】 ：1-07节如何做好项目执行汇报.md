# 04.【有说服力的工作汇报】 ：1-07节如何做好项目执行汇报

在本节课中，我们将学习如何在项目执行过程中，当遇到计划外的困难或问题时，进行清晰、有效的汇报，以推动项目继续前进。

上一节我们介绍了项目启动汇报。本节中，我们来看看项目进入执行阶段后，当出现计划外情况时，应如何进行汇报。

## 项目执行汇报的核心场景

项目执行汇报，特指在项目遇到计划外困难或问题时进行的汇报。这与定期的周期性汇报不同，其核心目标是**解决问题并推动项目前进**。

你可能会问：项目启动时不是已经定好了方案、计划和分工吗？怎么还会有计划外的问题？

确实，项目开始前我们会制定详尽的计划。但项目终究由人推动，只要涉及人，就存在不确定性。项目周期越长、涉及人员和部门越多，不确定因素也越多。此外，还有外部不可控因素，例如：
*   天气影响工程进度。
*   合作方预算临时变动。
*   市场环境突然变化。

计划永远赶不上变化。我们无法做到万无一失，只能学会有效应对。

## 项目执行汇报的三段式模型 🧱

遇到变化、问题和困难时，推荐使用以下汇报模型。该模型由三部分构成：
1.  **情况描述**
2.  **问题对策**
3.  **推进计划**

以下是每个部分的详细拆解。

### 1. 情况描述：为问题提供背景

在直接抛出问题前，需先对项目总体情况进行介绍和铺垫。这有助于汇报对象充分理解问题背景。

情况描述需包含两部分内容：

**项目现状**
描述项目当前进展到了什么程度，已取得哪些成果。
> **示例公式**：`项目现状 = 当前进度 + 已获成果`

**环境条件**
描述项目所处的客观环境与具备的条件。
> **示例公式**：`环境条件 = 外部客观因素 + 现有资源限制`

**示例**（接美妆店案例）：
“项目方案已开始落地，80%的门店完成装修，正在招聘推广人员，其中两家店人员已到位，可准备开业。”（此为**项目现状**）
“江浙地区连续降雨，门店所在商业街存在安全隐患，目前已暂停所有施工。”（此为**环境条件**）

清晰的情况描述，能帮助听众理解问题产生的背景、原因及其严重性。

### 2. 问题对策：界定问题并给出思路

说完背景，接下来进入核心的“问题对策”部分。

**阐述问题**
描述问题时，需对照原计划，并说明若不解决的后果。
> **示例代码框架**：
> ```
> 问题描述：
>   1. 指出偏离计划的具体点（原计划 vs. 现状）。
>   2. 阐明不解决的潜在后果（影响项目目标）。
> ```

**示例**（门店装修延期问题）：
“关于门店装修延期：原计划本月底完成A城市门店装修。受天气影响，目前进度已落后一周。”（**对照计划，指出问题**）
“若问题不解决，导致开业推迟，将错过下个月购物节，使新店宣传效果大打折扣，并影响后续运营。”（**阐明后果，凸显严重性**）

**提出对策思路**
得到重视后，需给出解决问题的初步思路，而非详细计划。目的是让汇报对象进行大致判断。
思路应以**更大目标为导向**，而非局限于眼前问题。

**示例**（针对上述装修问题）：
*   **思路A**：集中资源追赶装修进度，让项目回归正轨。
*   **思路B**：核心目标是确保门店按时开业。可同步推进招聘、物料制作等工作，不 solely 依赖装修完工。

### 3. 推进计划：明确行动与所需支持

最后一部分，需向汇报对象说明具体实施计划及所需资源支持。

**具体实施计划**
基于认可的对策思路，拆解为具体行动步骤。
> **示例公式**：`推进计划 = 具体行动1 + 具体行动2 + ...`

**所需资源支持**
明确执行计划需要哪些额外的资源、费用或跨部门配合。
> **示例公式**：`资源支持 = 额外人力/资金 + 所需协调与配合`

**示例**（接装修问题）：
“具体计划：1. 更换更专业的装修队，将两周工期压缩至一周；2. 即刻启动招聘，借用其他区域门店进行面试与前期培训，确保装修结束后人员马上到位。”（**具体实施计划**）
“所需支持：1. 因雨天施工，需更换防水材料，涉及额外费用；2. 需要其他门店配合招聘与培训工作，需提前协调。”（**资源支持**）

## 综合案例演练 📝

假设某公司员工小朱正在推进新产品线上宣传项目，遇到计划外情况，需进行汇报。

**情况描述**
*   **项目现状**：宣传文案与设计已按计划完成；已联系10个线上渠道，正逐一沟通合作方案。
*   **环境条件**：小型新媒体平台决策快，传统大型网站沟通周期较长。

**问题对策**
*   **问题**：重要宣传渠道X沟通数轮后方案仍迟迟未定。若无法合作，宣传效果将大打折扣；若继续等待，则影响宣传按时上线，进而影响后续销售。
*   **对策思路**：两害相权，确保宣传按时上线优先级更高。因此，双线并行：继续与X渠道沟通，同时立即接洽备选Y渠道。

**推进计划**
*   **具体行动**：
    1.  按原计划与其他渠道确认方案，保障按时上线。
    2.  员工小王继续跟进X渠道，可考虑适当妥协以促成合作。
    3.  员工小李立即联系Y渠道，争取达成合作。
*   **所需支持**：
    1.  为加速项目，需加派一人入组协助小王。
    2.  可能需要追加部分宣传费用，以保障方案执行。

## 课程总结

本节课我们一起学习了在项目执行遇阻时进行有效汇报的方法。

我们使用的核心汇报模型是：**情况描述 → 问题对策 → 推进计划**。
1.  **情况描述**需说清项目现状与环境条件，为问题铺垫背景。
2.  **问题对策**需对照计划指出问题、阐明后果，并提出以目标为导向的解决思路。
3.  **推进计划**需列出具体行动步骤，并明确所需的资源支持。

实际工作中的问题可能更复杂，但汇报的结构与方向万变不离其宗。掌握此框架，你便能清晰汇报，高效推动项目前进。
