# 课程04：有说服力的工作汇报 🎯
## 第06节：如何做好项目启动汇报 🚀

在本节课中，我们将学习项目型汇报的第一个关键场景：项目启动汇报。我们将了解其核心目标，并掌握构建一份清晰、有说服力的启动汇报所需的四个核心模块。

上节课我们拆解了周期性汇报的场景。接下来，我们将用三节课的时间来系统拆解项目型汇报。

所谓项目型汇报，是指工作汇报的全部内容都围绕某一个特定项目展开。例如：某客户品牌形象设计项目、某年度新员工培训项目、某品牌市场推广项目等。这类汇报不再简单地以时间为节点，而更多是以项目阶段为节点。

因此，在这三节课程中，我们将按照项目开始前、项目执行中以及项目完成后这三个阶段，把项目型汇报细分为项目启动汇报、项目执行汇报和项目总结汇报。我们将分别探讨这三个具体汇报场景的汇报目标是什么，以及汇报内容应如何梳理。

需要说明的是，这三节课分享的场景都属于项目已被批准之后要做的汇报。如果你有一个新想法或策划了一个新项目方案，但尚未被批准立项，可以参考后面关于提案建议如何汇报的课程。

本节课，我们先讲解项目启动汇报。

有的同学可能会问：项目还没开始，怎么就汇报了？什么都没干，汇报什么？没错，项目刚启动时的汇报，自然不是汇报“干了什么”，而是说明“要干什么”。

一个项目的顺利执行往往不是一个人的事，它会涉及跨部门协同，或需要多人参与，并且会持续很长时间。这就涉及到大量的资源协调。因此，必须在项目开始之前，就项目的相关人为何要参与、在项目中具体干什么、大家如何分工协作等问题进行说明。最关键的是让所有人都认同并愿意积极配合。

例如开篇提到的市场推广项目，除了牵头的市场部，设计、销售、公关等多个部门都需参与。这个项目与这些部门有何关联？需要参与多长时间？市场部自身需要在什么时间节点、由谁完成什么工作？出现问题后向谁汇报？项目启动汇报的目的就是提前解答这些问题。

项目启动汇报的目的是把该让大家知道的信息传递到位，并获得相关人员和执行团队的认可。如果这些信息没有传达到位，大家不能认可，就会出现扭捏不配合、半路撂挑子，或出了问题互相推诿等情况，项目就很难按计划顺利完成。

总之，项目启动汇报就是要把话说在前头。在项目开始执行之前，就要把与项目有关的一切信息传递给相关人员，并达成共识。

那么，项目启动汇报需要传递哪些信息并达成共识呢？我认为至少应包含四个模块：**目标意义**、**方案计划**、**职责分工**和**协作机制**。

我们逐一进行说明。

### 一、目标意义

第一个模块是目标意义，即这个项目最终要达到什么效果才算成功，以及为什么这个效果值得大家付出努力去争取。

这部分是项目启动汇报中非常重要的内容。把项目的意义和价值传递到位，项目的参与者才会有意愿、有动力积极配合，共同推动项目前进。

例如，在重要体育比赛前，教练会给运动员打气：“这场只要我们赢对手两个球，就能冲出亚洲、走向世界，创造历史，每个人都会被载入史册。”当然，这是个夸张的例子。我们的工作并非每个项目都有载入史册的意义，但肯定对公司或团队发展有重要意义。

例如，某个项目的目标可以这样描述：
> 我们这个市场推广项目的目标，是在未来半年内让用户数量增长20%。按照这个速度增长新用户，可以使我们的市场份额保持在行业前三。另一方面，这些新用户在未来2到3年，预计会给公司带来约2000万的利润。

这样描述，大家听起来就会很有干劲。

这里需要注意的是，目标一定要**具体、可量化、有时间节点**。换句话说，要让汇报对象和相关人员能够明确知道项目完成到什么程度就算成功。

### 二、方案计划

说完了目标意义，第二个模块是方案计划。即为了实现上述目标，我们这个项目具体要做哪些事，每件事需要在什么时间点完成。

这里最好为每一件事都设定一个明确的目标，也称为**里程碑**。仍以前面的市场推广项目为例：
> 为了实现新用户增长20%的目标，平均每月需新增用户6000个。为此，每月将安排一场针对新用户的促销活动，以及一次“老带新”营销宣传。其中，9月份是配合新品发布的活动A，目标是带来8000个新用户；10月份是活动B，目标是带来6000个新用户，以此类推。

汇报这些内容的本质，是让汇报对象相信，按照这个方案和计划执行，项目的目标能够达成。

### 三、职责分工

知道了需要做什么，还得知道需要谁来干。职责划分要清楚，分工要明确，项目的执行者和参与者才能各司其职，遇到问题也不至于相互推诿。

在职责分工部分，要配合方案计划的内容，让每个人都明确需要在什么时间节点交付一个什么结果。
例如：
> 在推广工作中，小张负责确认9月份宣传物料的制作。这就需明确在几月几号物料必须制作完成并配送到活动场地。

### 四、协作机制

最后一个模块是协作机制。所谓协作机制，是指沟通、商讨、汇报、决策的规则。例如：多长时间同步一次项目进展？通过什么方式同步？跨部门沟通由谁发起？拿不定主意时由谁协商？出现分歧时谁说了算？

通常情况下，协作机制中涉及几种角色：**责任人**、**执行者**和**协助者**。
*   **责任人**：全权负责某一块工作，拿主意时由其拍板，出了问题由其担当。
*   **执行者**：负责执行决议，并将执行中的问题反馈出来，同时也要负责想办法。
*   **协助者**：一方面协助执行者完成具体工作，另一方面积极配合信息沟通。

这几种角色之间的互相协作，就形成了一个固定的模式，这就是协作机制。

那么，协作机制与职责分工有何区别呢？
*   **职责分工**更侧重每个人具体要做什么工作，解决的是非常明确、固定的问题。
*   **协作机制**更侧重互相之间的配合与沟通模式，解决的是动态变化的问题。

这就好比足球场上，每个球员都有自己的固定位置（前锋、后卫），这是职责分工。但在固定位置之外，他们还需要通过喊话、手势、听从中场指挥、协助攻防转换等方式来动态配合，解决场上瞬息万变的问题，这些角色和配合方式就构成了场上的协作机制。

回到工作示例，在市场推广项目中，可能有负责活动策划、文案设计、宣传物料、客户跟进等不同小组。每个小组的工作第一责任人是小组长，这是**职责分工**。而“项目初期每周一次小组长会，临近活动时每天一次；跨组沟通由小组长负责”，这些就是**协作机制**。

总之，协作机制是解决项目中的动态问题，让团队配合更顺畅。

最后，我们通过一个小例子，把整个项目启动汇报串联起来，供你参考。

假设公司行政部门要牵头开展一个办公区装修项目，其项目启动汇报可以这样组织：

1.  **目标意义**：我们公司要在两个月内完成对整个办公区的装修，以达到可重新正常办公的状态。此次装修不仅会提升办公环境档次，还将增加30个工位、两个会议室和一个休息区，大大提升办公区利用率与大家的办公体验。

2.  **方案计划**：装修将分两个阶段进行，每阶段一个月，分别装修南侧和北侧办公区。装修哪个区域，该区域的同事就需搬到楼上临时办公区。每个阶段的装修工作会在两周内完成，之后两周用于通风、除甲醛和保洁。

3.  **职责分工**：本次项目由行政部门牵头，负责整个装修工作的协调与沟通。其他部门主要配合。其中，南侧的A、B、C部门需在几月几号搬离工位，并在几月几号搬回；北侧相关部门也需明确搬离与搬回日期。请各部门助理做好固定资产登记。此外，因这是一次企业形象提升，需要宣传部门负责整个项目的跟拍，并在项目完成后发布宣传稿。

4.  **协作机制**：各部门搬家工作的第一责任人是部门经理。具体搬家安排由行政部的小王与各部门助理协调。整个装修期间的沟通将在部门助理群进行，大家有任何需求或问题，可直接联系行政部的朱经理。

这就是一个完整的项目启动汇报示例。实际项目启动可能比此例更复杂，但相信通过这个小例子，大家已经能够了解如何有逻辑地讲清这四部分内容，以达成共识。

### 课程总结 🎓

本节课我们学习了项目型汇报的第一个场景：项目启动汇报。项目启动汇报就是要把话说在前头，在项目执行前将与项目相关的一切信息传递给相关人员，并达成共识。

汇报内容主要包括四部分：
1.  **目标意义**：明确项目成功的具体标准及其价值。
2.  **方案计划**：阐述为实现目标所需的具体行动与时间节点（里程碑）。
3.  **职责分工**：明确每项任务由谁负责，在何时交付何种结果。
4.  **协作机制**：建立沟通、决策与配合的规则，以应对动态问题。

因为有了共识，一个项目才能顺利开始，落地执行时才有章可循、有法可依。但这并非意味着执行过程中就不再需要管理。下节课，我们将探讨项目执行过程中遇到问题时该如何汇报，以推动项目继续前进。
