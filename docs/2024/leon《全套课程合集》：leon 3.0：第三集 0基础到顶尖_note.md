# 课程名称：Leon 3.0 第三集 - 从零基础到顶尖 🚀

在本节课中，我们将学习Leon 3.0的核心概念与基础操作。课程内容从最基础的定义开始，逐步深入到关键功能的实践应用，旨在帮助初学者构建完整的知识体系。

## 概述

Leon是一个强大的自动化工具。本节课程将解释Leon的基本工作原理。

它的核心是监听事件并触发响应。这个过程可以用一个简单的公式表示：**`事件 (Event) -> 条件 (Condition) -> 动作 (Action)`**。

当特定事件发生时，Leon会检查预设的条件是否满足。如果条件满足，它将执行相应的动作。

## 核心组件解析

上一节我们介绍了Leon的基本工作原理，本节中我们来看看构成这个工作流的三个核心组件。

### 1. 事件 (Event)

事件是启动整个流程的触发器。它可以是用户的某个操作、系统发出的信号或到达的特定时间。

以下是常见的事件类型示例：
*   用户发送了一条消息。
*   系统时钟到达了上午9点。
*   一个新的文件被添加到文件夹中。

### 2. 条件 (Condition)

条件是一个判断语句，用于筛选是否继续执行。只有满足所有条件，动作才会被触发。

条件通常表示为逻辑判断。例如，一个判断消息发送者的条件可以用如下伪代码描述：
```python
if message.sender == “管理员”:
    # 条件满足，继续执行
```

### 3. 动作 (Action)

动作是流程最终执行的任务。它是Leon为了响应事件而进行的实际操作。

以下是动作的一些例子：
*   向用户回复一条消息。
*   发送一封电子邮件。
*   运行一个特定的程序或脚本。

## 创建你的第一个自动化流程

理解了核心组件后，现在我们可以尝试将它们组合起来，创建一个简单的自动化流程。

假设我们想实现一个功能：当用户在聊天中说出关键词“你好”时，Leon自动回复“你好，有什么可以帮您？”。

以下是实现此流程的步骤：
1.  **设置事件**：选择事件类型为“接收新消息”。
2.  **添加条件**：设置条件为“消息内容包含‘你好’”。
3.  **定义动作**：设置动作为“发送消息”，内容为“你好，有什么可以帮您？”。

通过以上三步，一个基础的自动化回复流程就搭建完成了。你可以在此基础上，为事件添加更多条件，或让动作执行更复杂的任务。

## 总结

本节课中我们一起学习了Leon 3.0自动化工具的基础知识。我们首先了解了其 **`事件 -> 条件 -> 动作`** 的核心工作模型，然后分别探讨了事件、条件和动作这三个组件的作用与示例。最后，我们通过一个简单的“自动问候”流程，实践了如何将三者组合应用。掌握这些基础是构建更复杂、更强大自动化流程的关键。
