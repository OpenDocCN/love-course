# 课程：英雄联盟游戏机制解析 - P1 🎮

在本节课中，我们将要学习《英雄联盟》中一个关于“峡谷先锋”的特殊游戏机制与历史Bug。我们将解析其核心效果，并回顾一个曾出现的异常情况。

## 峡谷先锋的机制与改动 🏰

上一节我们介绍了课程主题，本节中我们来看看“峡谷先锋”这个游戏单位的核心机制。峡谷先锋是《英雄联盟》中的一种史诗级野怪，击败后队伍可以召唤它来协助推进防线。

其主要效果包含以下两点：
*   **对防御塔造成巨额伤害**：峡谷先锋的冲锋攻击能对防御塔造成基于其最大生命值百分比的伤害。
*   **为召唤者提供增益效果**：击败峡谷先锋的英雄会获得一个名为“先锋之眼”的消耗品，使用后可以召唤峡谷先锋。此外，该英雄在回城时会获得一个短暂的伤害减免效果。

用公式描述其核心推进作用可简化为：
**防御塔承受伤害 = 防御塔最大生命值 × 特定百分比**

## 历史上的异常Bug案例 ⚠️

了解了常规机制后，我们来看看一个历史上出现过的异常情况。在一次游戏版本更新后，峡谷先锋的机制发生了改动，其攻击防御塔的能力变得尤为突出。然而，在此次改动后，曾出现了一个严重的游戏错误。

以下是在一段游戏视频（BV1xp4y1B7go）中记录到的异常现象：
1.  一名使用英雄“萨科”（俗称小丑）的玩家，正常召唤了第一个峡谷先锋。
2.  随后，该玩家在极短时间内，**异常地再次召唤出了第二个峡谷先锋**。
3.  此Bug导致峡谷先锋可以被近乎无限地连续召唤，其场面犹如拥有无敌效果，对游戏平衡造成了巨大破坏。

这个Bug使得敌方防线在短时间内承受了远超设计预期的压力。

## 总结与回顾 📝

本节课中我们一起学习了“峡谷先锋”的基础机制及其一个著名的历史Bug。我们了解到峡谷先锋是强大的推进工具，拥有对塔高额伤害和提供减伤增益的效果。同时，我们也回顾了那个允许玩家异常重复召唤峡谷先锋的程序错误，它曾在短时间内极大地影响了游戏的正常进程。

理解这些机制与历史，有助于我们更全面地认识《英雄联盟》游戏的演变与发展。
